1. ВВЕДЕНИЕ
========================================
Информация является одним из ключевых ресурсов современного общества. С ростом объема цифровых данных и числа каналов их передачи возрастают требования к защите конфиденциальности, целостности и доступности данных. Для решения задач защиты информации широко используются криптографические и стеганографические методы.

Криптография предназначена для преобразования данных в форму, непонятную без знания ключа или специальных параметров. Стеганография решает иную задачу: сокрытие самого факта наличия сообщения. Совместное применение этих подходов позволяет повысить стойкость системы защиты: даже при обнаружении контейнера злоумышленнику необходимо дополнительно преодолеть криптографическую защиту.

Целью курсовой работы является разработка настольного программного средства SecureData_Suite, обеспечивающего:
- шифрование и дешифрование текстовой информации;
- шифрование и дешифрование файлов;
- сокрытие сообщений в изображениях методом LSB;
- локальное хранение зашифрованных копий (кеширование) для учета и восстановления результатов работы.

Разработанное приложение имеет графический интерфейс на базе Qt, поддерживает несколько алгоритмов и ориентировано на учебно-практическое применение.

3. АНАЛИЗ ПРЕДМЕТНОЙ ОБЛАСТИ И ВЫБОР ТЕХНОЛОГИЙ
========================================
3.1. Криптографические и стеганографические подходы
Классические шифры (Caesar, Vigenere, Atbash) просты в реализации и полезны для демонстрации базовых принципов. XOR, перестановочные методы и упрощенные блочные схемы дают практическое представление о побайтовой и блочной обработке данных. Алгоритмы Huffman и RSA расширяют функциональность: первый служит примером кодирования/компрессии, второй — примером асимметричного подхода.

Стеганографический метод LSB (Least Significant Bit) позволяет внедрять биты сообщения в младшие биты пикселей изображения, минимально искажая визуальное восприятие контейнера.

3.2. Обоснование выбора Qt и C++
Выбор C++ обусловлен высокой производительностью и широкими возможностями управления памятью и структурой приложения. Qt обеспечивает:
- кроссплатформенный подход к GUI;
- встроенные средства работы с файлами, строками, ресурсами и изображениями;
- систему сигналов/слотов для удобной организации событийной логики.

3.3. Среда разработки
- ОС: Windows 11
- Компилятор: MSVC 2022
- Фреймворк: Qt 6.10
- Система сборки: CMake

4. ПРОЕКТИРОВАНИЕ ПРОГРАММНОГО СРЕДСТВА
========================================
4.1. Модульная структура
Проект организован по директориям:
- src/crypto — реализации шифров и менеджер алгоритмов;
- src/stego — модуль LSB-стеганографии;
- src/ui — главное окно и привязка интерфейса;
- src/utils — кеширование, менеджер локализации и вспомогательные сервисы;
- include/... — заголовочные файлы по модулям;
- resources — ресурсы приложения (gif, qrc, переводы, алфавиты).

4.2. Ключевые классы
- CipherBase: базовый интерфейс для шифров.
- CipherManager: регистрация и выбор конкретного алгоритма по идентификатору.
- LsbSteganography: сокрытие/извлечение текста в изображении.
- FileCache: сохранение зашифрованных копий и метаданных.
- TranslationManager: загрузка/переключение языковых ресурсов.
- MainWindow: координация пользовательских сценариев через GUI.

4.3. Принцип расширяемости
Для добавления нового алгоритма достаточно:
1) унаследоваться от CipherBase;
2) реализовать encrypt/decrypt (и при необходимости бинарные методы);
3) зарегистрировать алгоритм в CipherManager.

========================================
6. ОПИСАНИЕ РЕАЛИЗОВАННЫХ АЛГОРИТМОВ
========================================
6.1. Caesar
Сдвиг символа на фиксированное число позиций. Поддерживается латиница и кириллица (алфавит вынесен в отдельный ресурс).

6.2. Vigenere
Полиалфавитный шифр с ключевым словом. Подходит для демонстрации зависимости шифрования от позиции символа.

6.3. Atbash
Зеркальная замена символов алфавита. Не требует ключа, используется как пример подстановочного метода.

6.4. XOR
Побайтовая операция XOR с повторяющимся ключом. Поддерживается для текста и бинарных данных, что позволяет применять алгоритм к файлам.

6.5. Permutation
Перестановка символов по ключу. В шифртекст включается префикс длины, позволяющий корректно восстановить исходные данные при обратном преобразовании.

6.6. Simple AES
Учебная реализация блочного шифрования по принципам AES (в упрощенном виде). Используется для демонстрации блочной криптографии и работы с padding.

6.7. Huffman
Кодирование на основе частот символов и бинарного дерева. В проекте используется как алгоритм кодирования/компрессии с заголовком и Base64-представлением результата.

6.8. RSA
Учебная асимметричная схема с генерацией пары ключей. Применяется для демонстрации различий между публичным и приватным ключами.

Примечание: часть алгоритмов реализована в учебных целях и не предназначена как промышленный криптографический стандарт.


========================================
7. РЕАЛИЗАЦИЯ СТЕГАНОГРАФИИ (LSB)
========================================
7.1. Суть метода
Метод LSB изменяет младшие биты цветовых каналов пикселей контейнера. Изменения минимальны и визуально часто неразличимы.

7.2. Реализованный алгоритм
- В начало контейнера записывается длина сообщения (фиксированный заголовок).
- Далее по битам записывается полезная нагрузка.
- При извлечении сначала читается длина, затем соответствующее количество бит.

7.3. Валидация вместимости
Перед внедрением выполняется проверка: сообщение должно помещаться в изображение с учетом служебного заголовка.

7.4. Возможные ошибки
- Попытка извлечения из обычного изображения, не содержащего скрытых данных.
- Неверная длина при поврежденном контейнере.
- Несоответствие формата файла.


========================================
8. РАБОТА С ФАЙЛАМИ И КЕШИРОВАНИЕМ
========================================
8.1. Шифрование файлов
Вкладка Files позволяет выбрать входной и выходной файл. Для бинарного режима применяются алгоритмы, поддерживающие побайтовую обработку.

8.2. Кеш зашифрованных копий
При включенной опции кеширования после шифрования создается копия зашифрованных данных в локальном каталоге приложения, а также метаданные (исходный путь, размер, алгоритм, дата).

8.3. Назначение кеша
- упрощение повторного доступа к результатам;
- учет выполненных операций;
- удобство контроля и восстановления в учебном сценарии.

========================================
10. ТЕСТИРОВАНИЕ И ВЕРИФИКАЦИЯ
========================================
10.1. Функциональные проверки
Проверялись сценарии:
- корректность шифрования и дешифрования текста;
- обработка неверных ключей;
- шифрование/дешифрование файлов;
- внедрение и извлечение стеганографического сообщения.

10.2. Round-trip тесты
Для каждого поддерживаемого алгоритма проверялось:
Исходный текст -> Encrypt -> Decrypt -> Исходный текст.

10.3. Проверка граничных случаев
- пустой ввод;
- большой объем данных;
- неподдерживаемые форматы файлов;
- извлечение из изображения без скрытого сообщения.

10.4. Результаты тестирования
Критических сбоев в штатных сценариях не обнаружено. Для ошибок ввода и файловых ошибок реализованы диагностические сообщения пользователю.


========================================
11. РУКОВОДСТВО ПОЛЬЗОВАТЕЛЯ
========================================
11.1. Шифрование текста
1) Открыть вкладку Cryptography.
2) Выбрать алгоритм.
3) Ввести исходный текст.
4) Ввести ключ (если требуется).
5) Нажать Encrypt.
6) Для обратного преобразования нажать Decrypt.

11.2. Скрытие сообщения в изображении
1) Открыть вкладку Steganography.
2) Выбрать Input image.
3) Указать Output image.
4) Ввести сообщение.
5) Нажать Embed.

11.3. Извлечение сообщения
1) На вкладке Steganography выбрать изображение-контейнер со скрытыми данными.
2) Нажать Extract.
3) Прочитать результат в поле Message.

11.4. Шифрование файла
1) Открыть вкладку Files.
2) Выбрать Input file и Output file.
3) Выбрать алгоритм, поддерживающий файлы.
4) Указать ключ.
5) Нажать Encrypt file.

11.5. Дешифрование файла
1) Выбрать зашифрованный файл как Input file.
2) Указать путь для результата.
3) Ввести тот же ключ.
4) Нажать Decrypt file.
